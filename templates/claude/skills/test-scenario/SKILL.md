# test-scenario — запуск фазы 1

Оркестрирует «медленный умный» проход по сценарию силами субагента `playwright-tester`.

## Аргумент
Имя сценария = имя папки в `scenarios/`. Например: `/test-scenario create-order`.
Если имя не передали — покажи список папок в `scenarios/` (кроме `_TEMPLATE`) и спроси, какой гнать.

## Шаги

1. **Найди/создай сценарий.**
   - Если `scenarios/<имя>/scenario.md` существует — прочитай его.
   - Если папки нет — предложи создать из шаблона (`scenarios/_TEMPLATE/scenario.md`) и
     попроси пользователя заполнить шаги, прежде чем запускать тест. Не выдумывай сценарий сам.
   - Проверь фронтматтер: обязательно `baseUrl` и `name`. Если `baseUrl` пуст — спроси у пользователя.

2. **Подготовь структуру и окружение.**
   - Убедись, что есть `scenarios/<имя>/reports/`.
   - Проверь, что настроен живой Playwright MCP-браузер (инструменты `mcp__playwright__*`).
     Если нет — подскажи добавить `.mcp.json` (см. README) и переоткрыть сессию. Он обязателен.
   - Проверь, что есть креды: `scenarios/auth.json` (или env
     `TEST_USER`/`TEST_PASS`). Если файла нет — скажи пользователю скопировать
     `scenarios/auth.example.json` → `scenarios/auth.json` и вписать user/pass.
     Сам не создавай: пароля ты не знаешь.

3. **Запусти субагента `playwright-tester`** (синхронно — результат нужен сразу):
   - **Claude Code:** Agent tool, `subagent_type: "playwright-tester"`, `run_in_background: false`.
   - **Cursor:** делегируй `/playwright-tester` (или явным упоминанием субагента).

   Промпт субагенту:
   ```
   Сценарий: <имя>. Файлы в scenarios/<имя>/.
   baseUrl из scenario.md: <baseUrl>.
   Пройди сценарий по своим правилам (живой Playwright MCP-браузер, устойчивые локаторы (роль → текст → CSS),
   исходники проекта не трогать), собери scenarios/<имя>/draft.js только из подтверждённых
   шагов, проверь его прогоном через node и напиши отчёт в scenarios/<имя>/reports/<дата_время>.md.
   Верни: итог, число пройденных шагов, статус логина, пути к draft.js и отчёту.
   ```

4. **Покажи результат пользователю (сюда, в чат):**
   - статус-борд по шагам (✅/⚠️/❌) из отчёта субагента;
   - путь к `draft.js` и к файлу отчёта (кликабельными markdown-ссылками);
   - итог: пройден полностью / частично / провал.

5. **Подскажи следующий шаг:**
   - Если ✅ или ⚠️ (есть что финализировать) → предложи `/finalize-scenario <имя>` (фаза 2).
   - Если ❌ на раннем шаге → предложи поправить `scenario.md`/локаторы или проверить, поднято ли приложение.

## Важно
- Ты сам приложение не деплоишь без явной просьбы — только проверяешь доступность `baseUrl`.
- Исходники тестируемого проекта не правятся ни на этом, ни на следующем шаге.
